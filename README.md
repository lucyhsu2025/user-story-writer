# User Story Writer
> 把軟體產品需求中的某個功能，轉換成工程師方便開發的 User Story 的 Claude Code Skill

它做的是語言轉化＋細化＋結構化：把「模糊的人話需求」拆解成「工程師或 AI 能直接接手執行的精準規格」。本質上是一鍵產出一份寫得很好、可複用的規格。

## 為什麼我做這個
我的角色是用戶研究／CMI 與產品經理。我最懂用戶的「為什麼」，但這個洞察常常停在報告裡，交到工程師手上時又變回模糊需求。所以我做這個工具，把我腦中的需求快速轉成工程師能直接開發、有明確驗收指標的規格。

## 解決的痛點
洞察停在報告裡，工程師拿到的是模糊需求，來回確認耗時。這個 skill 用「作為／我想要／以便」三段式，把「誰、要什麼、為什麼」講清楚，工程師不用猜，驗收標準也接得上。

## Before / After
- 以前：手寫一份含驗收標準與流程圖的 story，約 120 分鐘
- 現在：描述一句話需求 → 約 5 分鐘產出，人只需審核與補充

## 實際輸出
輸入一句話需求，就能生成完整 User Story，包含：三段式需求、Given/When/Then 驗收標準（含異常流程）、邊界情境、Mermaid 流程圖，以及一份「待專業補充」清單。

範例：酸奶訂閱功能 → [看完整輸出](examples/04-yogurt-subscription.md)

## 人機協作邊界
- AI 負責：發散、生成初稿，以及最後的結構化細化。
- 人負責：判斷與拍板，隨時中斷、補入真實用戶佐證與產業知識。
- 為什麼不能全交給 AI：核心是業務判斷。不同場景的適應性，需要懂用戶的人來補。AI 產出時會把推測的部分標註〔信心:低〕⚠️，不偽裝成事實。

## 這是一個可攜的 Agent Skill
它是開放標準的 SKILL.md，可在 Claude Code / Agent SDK 使用，不綁單一平台——這是一個加分的技術信號。

## 對應崗位
產品經理／用戶研究：把初始的產品 Brief 或功能構想，轉化為工程師可執行的產品規格。
